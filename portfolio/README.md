# 作品集（算法/数据方向）材料整理区

这是一个用于**研究生申请**的作品集 Markdown 工程骨架。你把每个项目的原始材料按本文档的规则放进来后，我会基于这些材料产出最终的项目页面与首页。

## 1. 目录结构（建议）

```
portfolio/
  index.md                      # 作品集首页（最终会生成）
  projects/
    _template.md                # 项目页模板（可复用）
    roboTRAC.md                 # 项目页：RoboTRAC（主打）
    picguard.md                 # 项目页：PicGuard（主打）
    breast-us-mtl.md            # 项目页：乳腺超声恶性诊断（多任务）
  assets/
    roboTRAC/                   # 该项目的图/表/截图（png/jpg/svg）
    picguard/
    breast-us-mtl/
  raw/
    roboTRAC/                   # 原始材料（不必公开的也放这）
    picguard/
    breast-us-mtl/
```

> 说明：`assets/` 放**作品集中会引用的图表**；`raw/` 放**更完整的原始材料**（日志、表格、长文档、代码压缩包等）。

## 2. 每个项目你需要提供的“原始材料包”（优先级从高到低）

### P0（最重要，缺了会影响写作）
- **项目说明**（哪怕是几段文字）：任务定义（输入/输出）、你负责的部分、评价指标
- **数据说明**：数据来源/规模/字段或标签定义、划分方式（train/val/test）
- **实验结果**：最终指标 + baseline 对比（表格/截图均可）
- **关键图**：至少 2–6 张（流程图、曲线图、对比图、可视化、bad case）

### P1（强烈建议）
- **方法细节**：核心思路/公式/模块图、为什么这么做
- **消融/对照**：你改动点带来的提升（哪怕只做 1–2 个也很好）
- **错误分析**：10–20 个典型 bad case（含简短解释）

### P2（加分项）
- 可复现信息：代码仓库/压缩包、requirements/conda env、随机种子、训练日志
- 产出证明：论文/专利/比赛证书、在投信息、部署链接与截图

## 3. 你把材料发我时的“最省事标注法”

对每个项目发一个消息块，按下面 4 行写开头（复制即可）：

```
【项目】RoboTRAC
【材料类型】数据/结果/图表/代码/说明（删掉不适用的）
【你最想强调】（例如：不平衡处理+召回提升、部署落地等）
【你不确定/想让我补写】（例如：Related Work、Limitations）
```

然后把对应文件直接附上（图片、表格、文档、压缩包或仓库链接都行）。
